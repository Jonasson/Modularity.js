Introduction

One challenge when developing JavaScript heavy web sites/applications is to organize them in a good way. Modularity.js approaches this problem with the help of a couple of great JavaScript projects and then add some own ideas on top of that.

About this documentation

This site is built with Modularity.js, and are in itself an example of how Modularity.js works. Please, take a look at the source if you are curious and want an example.

Just as the project itself, this documentation is in a rough state. Expect a lot of Swenglish and quirky behavior...

Declarative

The idea is to declare more things in your markup. This is an example:

    <html lang="en">
  <head>
    <title>Silly Example</title>
    .
    .
    .
    <script data-main="scripts/bootstrap.js" src="scripts/require.js"></script>
  </head>
  <!-- The routes below is handled by a Backbone.js controller -->
  <body id="controller" data-controller='{
    ""            : [ "welcome:article" ],
    "about"       : [ "about:article" ],
    "inspiration" : [ "inspiration:article" ]
  }'>
    <div id="page">
      <!-- this module turns the menu into a dropdown menu.
           It's unaffected by the controller. -->
      <nav data-menu='{ "effect": "slide" }'>
        <ul>
          <li><a href="#">Welcome</a></li>
          <li><a href="#about">About</a></li>
          <li><a href="#inspiration">Inpsiration</a></li>
        </ul>
      </nav>

      <!--
        These divs are bound to an article module. The controller decides if
        they should be visible or not.
        The module checks the div id to decide which content to load.
      -->
      <div id="welcome" data-article='{}'></div>
      <div id="about" data-article='{}'></div>
      <div id="inspiration" data-article='{}'></div>
    </div>
  </body>
</html>

Dependencies

The goal has been to minimize the dependencies to a minimum, but at this stage Modularity.js is dependent on these four libraries, all good in their own way and hard to be without.

  • RequireJS

    RequireJS allows you to divide your project into smaller parts in a clever way.

  • Backbone.js

    Backbone.js is a great project. Modularity.js adds a more modular thinking to the mix.

  • Underscore.js

    A collection of handy functions.

  • jQuery

    Great DOM manipulation.
